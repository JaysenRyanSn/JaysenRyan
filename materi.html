BAB 1. Kualitas Sistem

1.1 Pengertian Kualitas Sistem

Kualitas sistem adalah tingkat kemampuan suatu website dalam menyediakan fungsi yang stabil, mudah digunakan, cepat diakses, dan mampu bekerja sesuai kebutuhan pengguna. Dalam pengembangan website, kualitas sistem menjadi dasar penting karena memengaruhi apakah pengguna merasa nyaman atau justru terbebani saat menggunakan web tersebut.

1.2 Peran Kualitas Sistem dalam Pengembangan Website

Dalam konteks tugas pembuatan website, kualitas sistem berperan sebagai fondasi utama. Website yang tampilannya bagus belum tentu berkualitas jika sistemnya error, lambat, atau sulit digunakan. Karena itu, mahasiswa perlu memahami aspek teknis yang mendukung berjalannya sistem dengan baik.

1.3 Indikator Kualitas Sistem

Usability. Website harus mudah dipahami oleh pengguna baru. Menu, tombol, dan navigasi harus jelas serta tidak membingungkan. Contoh. menu navbar yang konsisten di setiap halaman.

Reliability. Sistem harus dapat berjalan stabil ketika digunakan. Tidak boleh sering error, halaman tidak boleh not found, dan fitur harus berfungsi setiap saat.

Efficiency dan Performance. Website sebaiknya dapat dibuka dengan cepat. Loading yang terlalu lama membuat pengguna kehilangan minat.

Security. Walau website tugas sederhana, keamanan tetap diperlukan.

Functionality. Semua fitur yang direncanakan harus berfungsi.

1.4 Penerapan Kualitas Sistem pada Website Proyek

Gunakan struktur kode yang rapi dan terorganisir.

Pastikan menu dapat diakses dari setiap halaman.

Lakukan pengecekan error melalui console browser.

Optimalkan gambar agar ukuran file tidak terlalu besar.

Uji website di HP dan laptop untuk memastikan kecepatan stabil.

1.5 Pentingnya Kualitas Sistem

Kualitas sistem yang baik meningkatkan.

Pengalaman pengguna.

Efisiensi penggunaan.

Tingkat kepuasan.

Kemungkinan pengguna kembali.

BAB 2. Kualitas Informasi

2.1 Pengertian Kualitas Informasi

Kualitas informasi adalah sejauh mana konten yang ditampilkan website benar, relevan, lengkap, dan tidak membingungkan pengguna.

2.2 Peran Kualitas Informasi dalam Pembuatan Web

Dalam pembuatan website, mahasiswa tidak cukup hanya fokus pada tampilan visual.

2.3 Indikator Kualitas Informasi

Akurasi.

Relevansi.

Kelengkapan.

Konsistensi.

Ketepatan waktu.

Objektivitas.

Kejelasan.

2.4 Penerapan Kualitas Informasi pada Website Proyek

Menuliskan paragraf yang jelas.

Menghindari teks dummy.

Melakukan pengecekan fakta.

2.5 Dampak Informasi yang Berkualitas

Meningkatkan pemahaman pengguna.

Meningkatkan kredibilitas website.

BAB 3. Kualitas Layanan

3.1 Pengertian Kualitas Layanan

Kualitas layanan merujuk pada pengalaman pengguna selama berinteraksi dengan website.

3.2 Peran Kualitas Layanan dalam Web

Layanan mencakup bagaimana pengguna menemukan informasi dan mendapatkan bantuan.

3.3 Indikator Kualitas Layanan

Responsiveness.

Assurance.

Empathy.

Tangibles.

Reliability.

3.4 Penerapan dalam Website Proyek

Gunakan warna yang tidak mencolok.

Tambahkan halaman bantuan.

Pastikan tautan berfungsi.

3.5 Pentingnya Kualitas Layanan

Pengguna lebih nyaman.

Tingkat kepuasan meningkat.

BAB 4. Penggunaan Sistem dan Kepuasan Pengguna

4.1 System Use

System Use menggambarkan seberapa sering dan seberapa lama Anda memakai sebuah website.

4.2 User Satisfaction

User Satisfaction menggambarkan perasaan puas setelah memakai website.

4.3 Hubungan

Kualitas sistem.

Kualitas informasi.

Kualitas layanan.

4.4 Penerapan

Memakai tampilan sederhana.

Memastikan website responsif.

4.5 Dampak

Pengguna ingin kembali.

Nilai proyek meningkat.
